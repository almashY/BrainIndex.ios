# BrainIndex 要件定義書

## 1. プロジェクト概要

### 1.1 アプリ名
BrainIndex

### 1.2 目的
学んだ知識を自分の言葉で言語化・記録し、タグによる関連付けと復習機能によってノウハウの定着を支援する。

### 1.3 背景
AI普及により情報取得が容易になった一方、知識として定着しにくいという課題が生じている。特に経験の浅いエンジニアは、調べた情報をそのまま利用するだけで自分の知識として蓄積しにくい。本アプリは「言語化」と「知識の関連付け」を習慣化することでこの課題を解決する。

---

## 2. 対象ユーザー

- 学習習慣を身につけたいエンジニア（主に初〜中級者）
- 知識を体系的に整理・定着させたいと考えている人

---

## 4. 画面定義

### 4.1 ホーム画面

**目的：** ノウハウ一覧の確認と新規作成への導線

**表示要素：**
- ノウハウ一覧（タイトル・タグ・作成日時）
- 今日のノウハウ作成数

**アクション：**
- 「ノウハウ作成」ボタン → 編集画面へ遷移
- ノウハウ一覧タップ → 詳細表示

---

### 4.2 編集画面

**目的：** ノウハウの新規作成・編集

**表示要素：**
- Markdownテキスト入力エリア
- Markdown入力補助ボタン（見出し、箇条書き、コードブロック等）
- タグ入力フィールド

**アクション：**
- Markdown入力補助ボタンタップ → 対応記法を挿入
- タグ入力 → タグを付与・管理
- 保存ボタン → ノウハウを保存してホーム画面へ戻る

---

### 4.3 復習画面

**目的：** 過去ノウハウのランダム表示による想起訓練

**表示要素：**
- ランダムに選ばれたノウハウの内容
- タグ一覧

**アクション：**
- 「次へ」ボタン → 別のノウハウをランダム表示
- 編集ボタン → 編集画面へ遷移

---

### 4.4 クイズ画面

**目的：** ノウハウをもとにした問題形式での理解度確認

**表示要素：**
- 問題文
- 選択肢（選択式）または入力フィールド（穴埋め式）
- 正誤フィードバック
- 元ノウハウへのリンク

**アクション：**
- 選択肢タップ / テキスト入力 → 回答
- 「次の問題へ」ボタン → 次の問題を表示

---

### 4.5 設定画面

**目的：** 通知設定などのアプリ設定管理

**表示要素：**
- 通知時間設定（DatePicker）
- 通知ON/OFFトグル

**アクション：**
- 通知時間設定 → 指定時間までにノウハウ未作成の場合に通知送信

---

## 5. 非機能要件

| 項目 | 要件 |
|---|---|
| プラットフォーム | iOS |
| データ保存 | ローカルDB（SwiftData）を使用 |
| オフライン対応 | オフラインでも全機能が利用可能 |
| 通知 | ローカル通知（UNUserNotificationCenter） |
| Markdown | 入力・プレビュー表示に対応 |

---

## 6. データモデル

### 6.1 ノウハウ（Knowhow）

| フィールド名 | 型 | 説明 |
|---|---|---|
| id | UUID | 一意識別子 |
| title | String | タイトル |
| content | String | 本文（Markdown） |
| createdAt | Date | 作成日時 |
| updatedAt | Date | 更新日時 |
| isFavorite | Bool | お気に入りフラグ |

### 6.2 タグ（Tag）

| フィールド名 | 型 | 説明 |
|---|---|---|
| id | UUID | 一意識別子 |
| name | String | タグ名 |

### 6.3 ノウハウ-タグ関連（KnowhowTag）

| フィールド名 | 型 | 説明 |
|---|---|---|
| knowhowId | UUID | ノウハウID（FK） |
| tagId | UUID | タグID（FK） |
